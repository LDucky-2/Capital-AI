## 1. Core User Tables

### `User_T`
Stores core authentication and authorization data for all system users.

| Column | Type | Description | Example |
| :--- | :--- | :--- | :--- |
| `User_ID` | INT | Primary Key. Unique system identifier. | `000000001` |
| `Name` | VARCHAR(100) | Full legal name of the user. | `Some guy` |
| `Password` | VARCHAR(30) | Plan text password (internal system). | `Someguy12345` |
| `Email_Address` | VARCHAR(80) | Unique contact email. | `someguy@gmail.com` |
| `Status` | VARCHAR(10) | Account state (Active, Frozen, Inactive). | `Active` |
| `Permission` | VARCHAR(30) | Role-based permission level. | `Management`, `Investor`, etc. |

## 3. Stock Market & Trading

### `Stock_T`
Defines active assets available in the market.

| Column | Type | Description | Example |
| :--- | :--- | :--- | :--- |
| `Stock_ID` | INT | Primary Key. | `000000001` |
| `Company_User_ID`| INT | FK to `Company_T`. Issuing company. | `102` |
| `Total_Shares` | INT | Number of shares issued at IPO. | `5000000` |
| `Current_Price` | DECIMAL | Live price in the market. | `152.75` |
| `Status` | VARCHAR | Trading status (Open, Closed). | `Open` |

### `Stock_Transactions_T`
Records all individual buy/sell actions.
- **Share Amount**: `250`
- **Type**: `buy` / `sell`

### `Trade_T`
Records Institutional asset trades.
- **Asset Type**: `Stock`, `Bond`, etc.
